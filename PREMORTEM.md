# PRE-MORTEM — 公開後に失敗したと仮定した検証

| 失敗要因 | 早期警告 | 予防策 |
|---|---|---|
| 「自社らしくない」と判断される | 参考LPへ戻す修正が増える | C案を視覚基準として差分を限定し、仕様書を共有 |
| 商品表示の根拠を問われる | 原材料・産地・価格への質問が増える | CLIENT_CONFIRMATION_CHECKLISTで承認管理 |
| LINE登録できない | CTAクリック後の離脱、誤リンク | iPhone／Android／PCで公式URLを実機検証 |
| スマホでFVが崩れる | 顔・器が切れる、見出しが4行以上 | 360／390／430pxで個別QA、3行固定を確認 |
| 表示が遅い | ヒーロー表示が2.5秒超 | next/image、Heroのみpriority、下部lazy、AVIF/WebP |
| AI感で信頼が落ちる | 手、ラベル、器、光に違和感 | 200%拡大監査。商品は公式画像優先 |
| 口コミが作為的に見える | 星・顔・過剰な属性追加を求められる | 原文と許可範囲を維持。架空要素を追加しない |
| CTAが多く圧迫感 | 固定CTAと本文CTAが同時に強く見える | PCはヘッダーのみ、スマホはHero通過後〜最終CTA前のみ |
| 修正範囲が膨張 | 3案混合や無制限修正を要求される | 納品形式・修正回数・原稿変更の扱いを契約前に定義 |
| コンペ案が流用される | URLが第三者へ共有される | noindex、透かし、審査後の公開停止 |
